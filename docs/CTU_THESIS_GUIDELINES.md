## 3. Formatting Standards

### 3.1 Headings

#### Part Headings
- **Format**: Centered, bold, uppercase, 14pt
- **Spacing**: 2cm above, 1cm below
- **Numbering**: None (visual separator only)
- **Example**: `PART 1: INTRODUCTION`

#### Chapter Headings (Level 1)
- **Format**: Centered, bold, uppercase, 14pt
- **Spacing**: 1.5cm above, 1cm below
- **Numbering**: `CHAPTER 1:`, `CHAPTER 2:`, etc.
- **Example**: `CHAPTER 1: BACKGROUND AND RELATED WORK`

#### Section Headings (Level 2)
- **Format**: Left-aligned, bold, uppercase, 13pt
- **Spacing**: 0.6cm above and below
- **Numbering**: `1.1`, `1.2`, etc.
- **Example**: `1.1 RESEARCH MOTIVATION`

#### Subsection Headings (Level 3)
- **Format**: Left-aligned, bold, sentence case, 13pt
- **Spacing**: 0.3cm above and below
- **Numbering**: `1.1.1`, `1.1.2`, etc.
- **Example**: `1.1.1 Problem Statement`

#### Sub-subsection Headings (Level 4)
- **Format**: Left-aligned, bold, sentence case, 13pt
- **Spacing**: 0.3cm above and below
- **Numbering**: `1.1.1.1`, `1.1.1.2`, etc.
- **Example**: `1.1.1.1 Technical Challenges`

### 3.2 Page Numbering

#### Front Matter
- **Style**: Lowercase Roman numerals (i, ii, iii, iv...)
- **Position**: Bottom center
- **Start**: From Table of Contents (page i)
- **Exclude**: Cover pages

#### Main Content
- **Style**: Arabic numerals (1, 2, 3, 4...)
- **Position**: Bottom center
- **Start**: From Part 1 Introduction (page 1)
- **Header**: Short thesis title (italic, 9pt) + horizontal line

### 3.3 Headers and Footers

#### Header (Main Content Only)
```
Short Thesis Title (max 50 characters, italic, 9pt)
_______________________________________________ (0.5pt line)
```

#### Footer (All Pages)
```
_______________________________________________ (0.5pt line)
                    Page Number
```

  4. Conclusions and implications (2-3 sentences)

**Keywords**:
- **Count**: 3-5 keywords
- **Format**: `Keywords: keyword1, keyword2, keyword3`
- **Position**: After abstract, separated by 1cm space

## 5. Main Content

### 5.1 Part 1: Introduction

**Required Sections**:

1. **Context and Problem Statement** (2-3 pages)
   - Research background
   - Problem identification
   - Motivation for research

2. **Related Work** (3-4 pages)
   - Literature review
   - Existing solutions
   - Research gap identification

3. **Objectives and Scope** (1-2 pages)
   - Research objectives (numbered list)
   - Scope definition (in-scope/out-of-scope)
   - Expected outcomes

4. **Research Content** (2-3 pages)
   - Research methodology
   - Technical approach
   - Development phases

5. **Thesis Outline** (1 page)
   - Structure overview
   - Chapter summaries

### 5.2 Part 2: Thesis Content

**Structure**: 3 chapters minimum

**Typical Organization**:

**Chapter 1: Background and Related Work** (10-15 pages)
- Theoretical foundations
- Technology overview
- Related research
- Comparative analysis

**Chapter 2: Design and Implementation** (15-25 pages)
- System architecture
- Database design
- Algorithm design
- Implementation details
- User interface design

**Chapter 3: Testing and Evaluation** (8-12 pages)
- Testing methodology
- Test cases and results
- Performance evaluation
- Usability testing
- Results analysis

### 5.3 Part 3: Conclusion and Future Work

**I. Conclusion** (3-5 pages)
- Summary of achievements
- Research contributions
- Limitations
- Lessons learned

**II. Future Work** (2-3 pages)
- Potential improvements
- Future research directions
- Recommendations

## 6. Back Matter

### 6.1 References

**Style**: IEEE Citation Style (mandatory)

**Format**:
```
[1] A. Author, "Article title," Journal Name, vol. X, no. Y, pp. 1-10, Year.
[2] B. Author, Book Title, Edition. City: Publisher, Year.
[3] C. Author, "Conference paper," in Proc. Conference Name, Year, pp. 1-5.
```

**Requirements**:
- Minimum 15 references
- At least 5 from recent years (last 5 years)
- Mix of journals, books, and conferences
- Properly cited in text using [1], [2], etc.

**Sorting**: Order of appearance in text

### 6.2 Appendices

**Numbering**: A, B, C...

**Common Content**:
- Survey questionnaires
- Interview transcripts
- Source code samples
- Additional data tables
- User manuals
- System documentation

**Format**:
```
APPENDICES

APPENDIX A: SURVEY QUESTIONNAIRE
[Content]

APPENDIX B: SOURCE CODE SAMPLES
[Content]
```

## 8. Figures and Tables

### 8.1 Figure Requirements

**Numbering**: Sequential within chapters (Figure 1.1, 1.2, 2.1, 2.2...)

**Caption Format**:
```
Figure 1.1. System Architecture Diagram
```

**Placement**:
- Centered on page
- Caption below figure
- 0.3cm space between figure and caption
- 0.6cm space before and after figure block

**Quality**:
- Minimum 300 DPI for images
- Vector graphics preferred (SVG, PDF)
- Clear, readable text in diagrams
- Professional appearance

**Referencing**: `As shown in Figure 1.1, the system...`

### 8.2 Table Requirements

**Numbering**: Sequential within chapters (Table 1.1, 1.2, 2.1, 2.2...)

**Caption Format**:
```
Table 2.1. Performance Comparison Results
```

**Placement**:
- Centered on page
- Caption above table
- 0.3cm space between caption and table
- 0.6cm space before and after table block

**Formatting**:
- Simple borders (no heavy lines)
- Header row in bold
- Left-align text, right-align numbers
- Consistent column widths

**Referencing**: `Table 2.1 shows the performance metrics...`

### 8.3 Equations

**Numbering**: Sequential within chapters (1.1, 1.2, 2.1...)

**Format**:
```
    E = mc¬≤                                    (1.1)
```

**Placement**:
- Centered
- Equation number right-aligned in parentheses
- 0.3cm space above and below

**Referencing**: `Equation (1.1) describes...`

## 9. Code and Technical Content

### 9.1 Code Listings

**Format**:
- **Font**: Courier New or Consolas, 10pt
- **Background**: Light gray (#F5F5F5)
- **Border**: 0.5pt gray border
- **Padding**: 0.3cm all sides
- **Line Spacing**: Single (1.0)

**Syntax Highlighting**: Recommended but not required

**Example**:
```python
def calculate_total(items):
    """Calculate total price of items."""
    return sum(item.price for item in items)
```

**Caption** (if needed):
```
Listing 2.1. Total Price Calculation Function
```
